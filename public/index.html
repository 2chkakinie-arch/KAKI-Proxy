The web, unblocked & encrypted.

KAKI Proxy is a next-generation web proxy built from scratch. It streams pages and video at native speed, hides the destination behind an encrypted token, and survives modern JavaScript apps — including YouTube playback with full DASH and Range support.

Tip: type any URL or just a domain. We'll encrypt it before redirect. No history, no logs, no third-party scripts.

Encrypted URLs

Every destination is wrapped in a random-nonce stream cipher keyed by your server secret. The address bar shows only a token.

True media streaming

Range requests, HLS playlists and DASH manifests are streamed byte-for-byte. YouTube plays without buffering hacks.

Service-worker safety net

If a script slips a raw URL into fetch or sets video.src directly, the worker catches it.

WebSocket bridge

Live chat, multiplayer and realtime apps tunnel through a single WS endpoint with full binary support.

Cookie isolation

Upstream cookies live in a server-signed jar, never leak to scripts, and survive cross-site navigation cleanly.

Zero-buffer pass-through

Binary responses are streamed without copy. HTML/CSS/JS rewriting only kicks in when it actually changes the bytes.

How it works

You enter a URL. The server encrypts it into a short token using an HMAC-derived keystream and a per-request 6-byte nonce, then 302s you to /p/<token>. For every following request, the server decrypts the token, opens an upstream connection, rewrites only the bytes that mention URLs (HTML attributes, CSS url(), HLS/DASH manifests) and streams the rest untouched. A tiny client bootstrap and a service worker catch anything that escapes static rewriting — fetch, XHR, WebSocket, video.src, history.pushState, dynamic Worker creation. Cookies are stored server-side inside a single kaki-jar cookie keyed by upstream host. Nothing else is persisted.
